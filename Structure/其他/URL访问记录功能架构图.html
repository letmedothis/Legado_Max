URL访问记录功能架构

Legado Plus · 网络请求追踪系统

🌐
Network Layer
网络层 - 请求拦截与记录
UrlRecordInterceptor
OkHttp拦截器，拦截所有网络请求并记录详细信息。通过AppConfig.recordUrl控制开关。
object Interceptor CoroutineScope
scope.launch { appDb.urlRecordDao.insert(record) }
HttpHelper
HTTP客户端配置中心，注册UrlRecordInterceptor到OkHttp客户端。
object OkHttpClient
builder.addInterceptor(UrlRecordInterceptor)
异步写入
💾
Data Layer
数据层 - 实体定义与数据访问
UrlRecord
URL记录实体类，包含请求URL、域名、方法、响应码、耗时、来源等字段。
@Entity tableName: url_records @Index: timestamp, domain
UrlRecordDao
数据访问对象，提供CRUD操作和Flow查询接口。
@Dao flowAll() flowSearch() flowByDomain()
AppDatabase
Room数据库，版本93新增url_records表。
@Database v93 @AutoMigration 92→93
StateFlow观察
🎨
UI Layer
界面层 - 状态管理与视图展示
UrlRecordUIState
UI状态密封类，封装Loading/Success/Error/Empty四种状态。
sealed class Loading Success Error Empty
UrlRecordViewModel
视图模型，管理UI状态、域名列表、搜索筛选逻辑。
BaseViewModel StateFlow<UIState> clearAll()
UrlRecordActivity
主界面Activity，实现搜索监听、域名筛选菜单、记录清除功能。
VMBaseActivity SearchView.OnQueryTextListener
UrlRecordAdapter
RecyclerView适配器，展示URL记录列表项，支持状态码颜色标识。
RecyclerView.Adapter ViewBinding
⚙️
Config Layer
配置层 - 开关控制与持久化
AppConfig
应用配置管理，recordUrl属性控制URL记录开关。
object recordUrl: Boolean
PreferKey
SharedPreferences键常量定义。
object const recordUrl

🔍
实时搜索
支持搜索URL、域名、来源名称，输入即搜索，响应式更新结果。
🏷️
域名筛选
动态生成域名菜单，按域名分类查看请求记录，单选模式。
🧹
智能清理
支持清除7天前、30天前或全部记录，显示删除数量确认。
⚡
异步记录
使用协程异步写入数据库，不阻塞网络请求，性能无影响。
🎨
状态可视化
响应码颜色标识：绿色成功、橙色异常、红色错误，一目了然。
📊
耗时统计
记录每个请求的耗时毫秒数，便于性能分析和问题排查。
