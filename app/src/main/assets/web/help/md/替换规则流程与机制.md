# 替换规则流程与机制

## 概述

app中存在**两种替换规则**，在不同阶段对正文内容进行净化处理：

1. **书源替换规则** - 获取正文时执行（入库前）
2. **独立替换规则** - 阅读显示时执行（显示前）

**核心原则：书源规则先执行，独立规则后执行，两者叠加生效。**

---

## 一、书源替换规则（replaceRegex）

### 定义位置
书源配置 → 正文处理规则 → `replaceRegex` 字段

### 语法格式
```
##正则匹配##替换内容      （全局替换）
##正则匹配##替换内容###  （仅替换第一个匹配）
```

### 执行时机
在 [BookContent.analyzeContent](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/model/webBook/BookContent.kt#L196-L203) 中执行：

```kotlin
// 全文替换（在保存内容到数据库之前）
val replaceRegex = contentRule.replaceRegex
if (!replaceRegex.isNullOrEmpty()) {
    contentStr = contentStr.split(AppPattern.LFRegex).joinToString("\n") { it.trim() }
    contentStr = analyzeRule.getString(replaceRegex, contentStr)
}
```

### 流程位置
```
抓取HTML → 提取正文 → HTML格式化 → 书源replaceRegex替换 → 保存到数据库
```

## 二、独立替换规则（ReplaceRule）

### 规则结构
[ReplaceRule](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/data/entities/ReplaceRule.kt#L24-L60) 核心字段：

| 字段 | 说明 |
|------|------|
| `name` | 规则名称 |
| `pattern` | 匹配内容（正则或普通文本） |
| `replacement` | 替换为 |
| `scope` | 作用范围（书籍名列表） |
| `scopeTitle` | 是否作用于标题 |
| `scopeContent` | 是否作用于正文 |
| `excludeScope` | 排除范围 |
| `isRegex` | 是否使用正则 |
| `timeoutMillisecond` | 超时时间（默认3000ms） |

### 执行入口
[ContentProcessor.getContent](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/help/book/ContentProcessor.kt#L92-L222)

### 完整处理流程

```
ContentProcessor.getContent 执行流程
  │
  ├─ 1. 去除重复标题
  │     检测并移除正文开头的章节标题
  │
  ├─ 2. 重新分段（可选）
  │     按配置重新组织段落结构
  │
  ├─ 3. 简繁转换
  │     根据设置进行简繁互转
  │
  ├─ 4. HTML内容保护
  │     特殊HTML用占位符替换，防止被破坏
  │
  ├─ 5. 执行替换规则 ⭐
  │     ├─ 每行trim去首尾空白
  │     ├─ 依次遍历所有启用的规则
  │     ├─ 正则模式：带超时检测的替换
  │     ├─ 普通模式：直接String.replace
  │     ├─ 记录生效的规则列表
  │     └─ 正则超时自动禁用该规则
  │
  ├─ 6. 还原HTML占位符
  │
  ├─ 7. 添加标题 + 标题替换规则
  │     对章节标题应用规则后添加到正文开头
  │
  └─ 8. 分段渲染
        去除空行、添加缩进，返回UI显示
```

### 核心替换逻辑

[ContentProcessor](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/help/book/ContentProcessor.kt#L157-L191) 实现：

```kotlin
// 每行trim
mContent = mContent.lines().joinToString("\n") { it.trim() }

// 依次应用规则
getContentReplaceRules().forEach { item ->
    val tmp = if (item.isRegex) {
        // 正则替换：支持@js:脚本、超时检测
        mContent.replace(
            item.name, item.regex, item.replacement,
            item.getValidTimeoutMillisecond(), chapter, replaceBook
        )
    } else {
        // 普通文本替换
        mContent.replace(item.pattern, item.replacement)
    }
    // 只有实际生效才记录
    if (mContent != tmp) {
        effectiveReplaceRules.add(item)
        mContent = tmp
    }
}
```

### JavaScript替换支持

replacement 支持 `@js:` 前缀执行JavaScript：

```
@js:result.replace(/abc/g, 'xyz')
```

其中 `result` 是匹配到的文本，`chapter` 是章节对象，`book` 是书籍对象。

### 超时保护机制

[RegexExtensions](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/utils/RegexExtensions.kt#L27-L89) 实现：

- 在IO线程执行正则替换
- 超过设定时间（默认3秒）则中断
- 正则超时自动禁用该规则并写入数据库
- 超时3秒仍未结束则重启应用

## 三、两种规则对比

| 特性 | 书源替换规则 | 独立替换规则 |
|------|-------------|-------------|
| 定义位置 | 书源配置 | 替换净化功能 |
| 语法 | `##match##replace` | 正则或普通文本 |
| 执行时机 | 获取正文时（入库前） | 阅读时（显示前） |
| 作用范围 | 该章节 | 可指定书籍/排除书籍 |
| JS支持 | 书源规则语法 | `@js:`前缀 |
| 超时检测 | 无 | 有（默认3秒） |
| 每本书开关 | 不可单独开关 | 可独立开关 |

## 四、替换顺序与组合流程

### 场景1：书源和替换规则界面都用了替换

**执行顺序：书源先替换 → 独立规则后替换**

```
原始网页内容
    ↓
【书源replaceRegex替换】← 书源配置的##match##replace
    ↓
保存到数据库（已净化一次）
    ↓
阅读时加载内容
    ↓
【独立替换规则替换】← 替换净化界面配置的规则
    ↓
显示给用户（已净化两次）
```

**示例：**
- 书源规则：`##广告##` （删除"广告"二字）
- 独立规则：`##重复内容##` （删除"重复内容"四字）

原始文本：`这是广告内容，这是重复内容内容`
1. 书源替换后：`这是内容，这是重复内容内容`
2. 独立替换后：`这是内容，这是内容`

### 场景2：书源正文+书源替换规则+独立替换规则

完整流程包含三个替换环节：

```
原始网页HTML
    ↓
① 提取正文内容（bookSource.ruleContent）
    ↓
② HTML格式化（HtmlFormatter）
    ↓
③ 书源replaceRegex替换 ← 第一层替换
   例如：##<[^>]+>##     （删除HTML标签）
    ↓
保存到数据库
    ↓
阅读时加载内容
    ↓
④ 去除重复标题
    ↓
⑤ 简繁转换
    ↓
⑥ HTML内容保护
    ↓
⑦ 独立替换规则替换 ← 第二层替换
   例如：逐条应用ReplaceRule列表
    ↓
⑧ 还原HTML
    ↓
⑨ 添加标题（可应用标题替换规则）
    ↓
⑩ 分段渲染显示
```

**关键点：**
- 书源replaceRegex在**入库前**执行一次
- 独立ReplaceRule在**显示前**执行一次
- 两者互不影响，按顺序叠加生效

### 场景3：##正则替换 与 @js: 组合使用

在独立替换规则中，`##` 和 `@js:` 是**同一规则的不同语法**，不是分开的两个步骤。

#### 书源中的 ## 语法

书源的 `##match##replace` 语法在 [AnalyzeRule.replaceRegex](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/model/analyzeRule/AnalyzeRule.kt#L482-L506) 中处理：

```kotlin
private fun replaceRegex(result: String, rule: SourceRule): String {
    if (rule.replaceFirst) {
        /* ##match##replace### 仅替换第一个匹配 */
        return if (matcher.find()) {
            matcher.group(0)!!.replaceFirst(regex, replacement)
        } else { "" }
    } else {
        /* ##match##replace 全局替换 */
        return result.replace(regex, replacement)
    }
}
```

#### 独立替换规则中的 @js: 语法

独立替换规则的 `@js:` 在 [RegexExtensions.replace](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/utils/RegexExtensions.kt#L37-L63) 中处理：

```kotlin
val isJs = replacement.startsWith("@js:")
val replacement1 = if (isJs) replacement.substring(4) else replacement

while (matcher.find()) {
    if (isJs) {
        // 执行JavaScript，result=匹配文本
        val jsResult = RhinoScriptEngine.eval(replacement1, bindings)
        matcher.appendReplacement(stringBuffer, jsResult)
    } else {
        matcher.appendReplacement(stringBuffer, replacement1)
    }
}
```

#### 组合使用流程

如果同时使用 `##` 和 `@js:`，流程如下：

```
书源replaceRegex规则：
##<script[^>]*>.*?</script>##     ← 删除脚本标签

      ↓ 保存到数据库

独立替换规则列表（按顺序执行）：
规则1: pattern="广告" → replacement="删除"      ← 普通替换
规则2: pattern="\d+"  → replacement="@js:parseInt(result)*2"  ← JS替换
规则3: pattern="重复" → replacement=""          ← 普通替换

执行流程：
原始内容："广告<script>alert(1)</script>123重复"
    ↓
【书源##替换】删除script标签
    ↓
"广告123重复"
    ↓
【独立规则1】普通替换：广告→删除
    ↓
"删除123重复"
    ↓
【独立规则2】正则匹配\d+，对每个匹配执行JS：123→246
    ↓
"删除246重复"
    ↓
【独立规则3】普通替换：重复→空
    ↓
"删除246"
    ↓
最终显示
```

#### @js: 执行细节

当 replacement 以 `@js:` 开头时：

```kotlin
// 1. 提取JS代码
@js:result.toUpperCase()
// 提取后：result.toUpperCase()

// 2. 为每个匹配执行JS
while (matcher.find()) {
    // result = 当前匹配的文本
    // chapter = 章节对象
    // book = 书籍对象
    // java = 扩展工具类
    val jsResult = RhinoScriptEngine.eval("result.toUpperCase()", bindings)
    matcher.appendReplacement(stringBuffer, jsResult)
}
```

**示例：**
```
pattern: \d+
replacement: @js:'[' + result + ']'

输入: "价格123元，数量45个"
输出: "价格[123]元，数量[45]个"
```

## 五、总结

### 替换优先级

```
获取阶段（入库前）：
  书源replaceRegex（##match##replace语法）

阅读阶段（显示前）：
  独立ReplaceRule列表（按顺序逐条执行）
  ├─ 普通文本替换
  ├─ 正则替换
  └─ JS替换（@js:前缀）
```

### 核心原则

1. **书源规则先于独立规则执行**
2. **独立规则按列表顺序依次执行**
3. **每次替换的结果作为下次替换的输入**
4. **JS替换在正则匹配基础上，对每个匹配结果执行JS**
5. **两种规则叠加生效，互不冲突**
